**Last Success:** Merged feature/admin-bearer-auth into main (246 tests passing).
**Next Step:** Feature complete — no pending work
**Context Note:** 246 tests passing on main. CredentialStore.cs deleted; AdminAuth.cs added; 20 new AdminBearerAuthTests.cs tests. Fixed TryValidateSessionCookie to use invite.FriendlyName.

### Approved Plan

**Branch:** `feature/admin-bearer-auth`

#### Removed

- `--credentials-file` and `--password` CLI flags
- `src/CredentialStore.cs` (entire file)
- Credential store tests

#### New CLI flags

- `--admin-username` (default: `"admin"`)
- `--admin-password` — if omitted: generate random 16-char password, write to `filebeam-admin.key` in CWD, print to terminal; on restart without flag read key file so password survives; `FILEBEAM_ADMIN_PASSWORD` env var takes precedence over both

#### Auth middleware order

1. Resolve admin password: env var → CLI flag → `filebeam-admin.key`
2. Basic Auth: match admin username + password → role `"admin"`
3. Bearer token: `Authorization: Bearer {id}` → `inviteStore.TryGet(id)` (NOT TryRedeem — no UseCount increment) → apply invite role/user
4. Session cookie: unchanged

#### HtmlRenderer — create modal

After creation show two copy fields:

- Browser: `https://host/join/{id}` — existing
- CLI: `Bearer {id}` — new (copies full header value)

#### HtmlRenderer — invite table rows

- 🔗 button: copies browser join URL (unchanged)
- ⌨ button (new): copies `Bearer {id}` (full header value)
- Inline note below table: _"The Bearer token and join link share the same ID — sharing the link grants both browser and API access."_

**Last Success:** Merged feature/audit-log-viewer into main (282 tests passing).

**Context Note:** 282 tests passing. GET /admin/audit added; RenderAuditLog + BuildNavLinks hasAuditLog param; TryParseAuditEntry; 14 new tests.

### Approved Plan

**Branch:** `feature/audit-log-viewer`

#### Files to change

| File                                          | Change                                                                                                                  |
| --------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| `src/RouteHandlers.cs`                        | Add `auditLogPath` constructor param; add `GetAuditLog` handler (admin-only)                                            |
| `src/HtmlRenderer.cs`                         | Add `RenderAuditLog(entries, navLinks)`; add `hasAuditLog` bool param to `BuildNavLinks`                                |
| `src/Program.cs`                              | Pass `auditLogPath` to `RouteHandlers`; register `GET /admin/audit`; thread `hasAuditLog` through `BuildNavLinks` calls |
| `tests/FileBeam.Tests/AuditLogViewerTests.cs` | New test file (~8 tests)                                                                                                |
| `README.md`                                   | Document `GET /admin/audit` endpoint                                                                                    |

#### Behavior

- Admin-only — non-admin → 403
- 404 when `--audit-log` not configured or is `-` (stdout, unreadable)
- Reads last 100 lines of NDJSON file (`File.ReadAllLines → TakeLast(100).Reverse()`), newest-first
- Malformed JSON lines silently skipped
- Graceful empty state when log file doesn't exist yet
- `<meta http-equiv="refresh" content="30">` auto-refresh
- Columns: Timestamp · Action · User · File · Bytes · IP · Status
- Nav link "Audit Log" rendered for admin users when log is configured

#### Tests (~8)

1. 404 when `auditLogPath` is null
2. 404 when `auditLogPath` is `-`
3. 403 for non-admin with configured log
4. 200 + HTML table for admin when log file has entries
5. Entries shown newest-first (reversed)
6. Malformed JSON lines are skipped
7. Empty table when log file is missing/empty
8. `BuildNavLinks` includes Audit Log link when `hasAuditLog=true` + admin role

---

#### Security decisions

- `TryGet` (not `TryRedeem`) for Bearer — UseCount only reflects browser join events
- `FILEBEAM_ADMIN_PASSWORD` env var avoids password in process list
- `filebeam-admin.key` persists auto-gen password across restarts
- `adminPassword` excluded from config export (intentional)
- README notes: TLS recommended for Bearer; shared-ID warning for invite links
