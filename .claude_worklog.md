### Phase 2: Invite system backend
- [x] Plan Approved
- [x] Branch Created | [x] Implementation | [x] Testing | [x] Documentation | [x] Merged

**Last Success:** Merged feature/invite-system into main (206 tests passing). README updated.
**Deliverables:** `src/InviteStore.cs`, `--invites-file` flag, sessionKey, CSRF header support, admin invite endpoints, `/join/{token}`, 24 new tests.

**Scope:**
- `InviteToken` record: Id (12-char URL-safe token), FriendlyName, Role, ExpiresAt (nullable), UseCount, CreatedBy, IsActive, CreatedAt
- `InviteStore` class: in-memory ConcurrentDictionary; optional persistence via `--invites-file <path>` (JSON, auto-save on create/revoke/edit, auto-load on startup)
- `--invites-file` flag added to Program.cs (parsed + forwarded to RouteHandlers)
- Endpoints: `POST /admin/invites` (JSON + X-CSRF-Token), `GET /admin/invites` (list JSON), `DELETE /admin/invites/{id}` (X-CSRF-Token), `PATCH /admin/invites/{id}` (JSON + X-CSRF-Token)
- CSRF middleware updated to also accept `X-CSRF-Token` header (DELETE/PATCH and JSON POST)
- sessionKey: 32 random bytes at startup; passed to RouteHandlers for cookie signing
- `GET /join/{token}` — validates token, sets signed `fb.session` cookie (HMAC-SHA256), increments UseCount, redirects to `/`
